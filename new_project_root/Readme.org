** Scientific raw-data immutability (recommended)
Use three layers:
1. Repository policy: treat `data/raw/` as immutable.
2. Filesystem: make `data/raw/` read-only.
3. Git hook: reject commits that modify `data/raw/**` unless explicit override is provided.

#+begin_src sh :results output :exports both
# Layer 2
ga_raw_lock data/raw

# Layer 3
ga_raw_hook_install data/raw

# If an intentional exception is needed:
GA_ALLOW_RAW=1 git commit -m "intentional raw data correction"
#+end_src

For already present files, a stronger filesystem lock is possible with the immutable bit:
#+begin_src sh :results output :exports both
# stronger lock (example)
chattr +i data/raw/*.tif

# to allow a deliberate change later
chattr -i data/raw/*.tif
#+end_src
